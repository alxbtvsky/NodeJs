NodeJs

by Alex Batovsky

Posted on February 10, 2015 at 9:00 PM

Introduction:

In this post I will be talking about the asynchronous, event-driven framework built on top of Chrome's JavaScript enginge, V8, that was designed for creating scalable network applications. NodeJs is a living creature, it is a set of libraries that are built from the ground up that help you focus on doing networking correctly. NodeJs is single-threaded and built on a concurrency model based on an event loop. NodeJs excels in building I/O bound applications with a multitude of concurrent connections. If you are looking to build a CPU-heavy application, you may want to look elsewhere. A few examples of great applications that can be built using NodeJs are single page applications, JSON APIs, and data-streaming applications.

Digging Deeper:

If you are reading this then it must mean that you are interested in using NodeJs to build your application. The remainder of this blog will be focused on how building an application on NodeJs will work opposed to what you may be used to. A huge difference between NodeJs and more traditional server side languages is that NodeJs never stops. All libraries in NodeJs have been built from the ground up to be non-blocking. NodeJs excels at idling, which will set your CPU to 0. While idling, NodeJs allows you to perform other tasks while waiting to be notified when the response is available. What this means is that Node will never wait while a url is being fetched. Instead, it registers a callback and lets the program continue. This allows massive scalability because it can handle concurrent operations without multiple threads. NodeJs does not buffer data into memory, instead it outputs it by chunks. When NodeJs has nothing left to do, it will exit naturally.
